# Devman Review Bot
Скрипт для автоматического отслеживания статуса проверки работ на образовательной платформе Devman.
Бот работает в режиме Long Polling: он «висит» на связи с сервером API и мгновенно присылает уведомление в Telegram,
как только преподаватель проверит ваш урок.

## Как это работает
Скрипт делает запрос API Devman, используя метку времени (timestamp).
Если за время ожидания статус работы изменился, бот присылает вердикт:
принята работа или отправлена на доработку, а также прямую ссылку на урок.

## Основные фишки:
- Отказоустойчивость: корректная обработка сетевых ошибок и таймаутов без падения скрипта.
- Надежность: уведомление не потеряется, если Telegram временно недоступен.
- Логирование: раздельные уровни отладки (DEBUG, INFO) для удобного мониторинга работы.

## Как установить
Скачайте репозиторий.
Убедитесь, что установлен [Python 3.13](https://www.python.org/downloads/release/python-3130/).

Для изоляции проекта рекомендуется использовать `.venv` виртуальное окружение:
```shell
python -m venv .venv
```

Активируйте виртуальное окружение:

**Windows**
```shell
.venv\Scripts\activate
```
**Linux/macOS**
```shell
source .venv/bin/activate
```

Затем используйте `pip` для установки зависимостей:
```shell
pip install -r requirements.txt
```

## Переменные окружения
Для работы с переменными окружения переименуйте файл `.env.example` в`.env`.

В файле находятся следующие переменные:
- `DEVMAN_API_TOKEN`=api_токен_девмана
- `TG_TOKEN`=токен_вашего_телеграм_бота
- `TG_CHAT_ID`=ваш_chat_id_в_телеграм
- `LOG_LEVEL`=какой_уровень_логгеров_вы_хотите_видеть

## Запуск
```shell
python watcher.py
```
